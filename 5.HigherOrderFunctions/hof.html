Complex Programs

A large program is a costly program, not because of the time it takes to build. Size almost always involves complexity and complexity confuses programmers . Confused programmers inturn introduces(bugs) mistakes into programs .A large program then provides a lot of space for these bugs to hide , making them hard to find.

Abstractions

Abstractions give us the ability to talk about problems at a higher (or more abstract) level, without getting sidetracked by uninteresting details.

Plain functions are a good way to build abstractions.

It is common for a program to do something a given number of times.
